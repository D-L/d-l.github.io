nim_closure闭包

Fri Jul 29, 2016 in using tags

Nim中的闭包

比较下下面的两段代码的不同:

a.nim

proc test(b : seq[int]) =
  proc inner() =
    if len(b) > 10:
      echo ">10"
    else:
      echo "<10"
  inner()

var a = @[1,2,3,4]
test(a)

b.nim

proc test(b : seq[int]) =
  proc inner(c : seq[int]) =
    if len(c) > 10:
      echo ">10"
    else:
      echo "<10"
  inner(b)

var a = @[1,2,3,4]
test(a)

结语:

由此也可以看出　在不需要的时候　尽量少用Closure 性能差异太大了的. 特别是传入的参数越庞大越复杂就性能越差!
